## 一、 概述

动态图标与状态展示是 Claude Code TUI 中最复杂的视觉系统。加载图标（Spinner）通过字符帧动画、微光扫过效果、停滞渐变红色等多层动画反馈 LLM 的工作状态；同时一套语义化的 Unicode 图标常量服务于权限提示、工具状态、媒体控制等场景。本文深入分析 Spinner 的帧动画机制、状态机切换逻辑、停滞检测算法，以及全部图标常量的定义与用途。

Spinner 系统源码位于 [`claude-code-source/src/components/Spinner/`](../../claude-code-source/src/components/Spinner/) 目录，图标常量位于 [`claude-code-source/src/constants/figures.ts`](../../claude-code-source/src/constants/figures.ts)。

## 二、 Spinner 字符帧动画

### 1. 默认字符集

Spinner 的动画帧由 `getDefaultCharacters()` 函数根据平台返回，定义在 [`claude-code-source/src/components/Spinner/utils.ts`](../../claude-code-source/src/components/Spinner/utils.ts#L4-L11)：

```typescript
// claude-code-source/src/components/Spinner/utils.ts#L4-L11
export function getDefaultCharacters(): string[] {
  if (process.env.TERM === 'xterm-ghostty') {
    return ['·', '✢', '✳', '✶', '✻', '*'] // Ghostty 用 * 替代 ✽
  }
  return process.platform === 'darwin'
    ? ['·', '✢', '✳', '✶', '✻', '✽']
    : ['·', '✢', '*', '✶', '✻', '✽']  // 非 macOS 用 * 替代 ✳
}
```

6 个基础帧字符按复杂度递增，模拟"思考汇聚"的视觉感：

| 帧序 | 字符 | Unicode | 含义 |
|---|---|---|---|
| 0 | `·` | `\u00b7` | 中点（最简） |
| 1 | `✢` | `\u2722` | 四角星 |
| 2 | `✳`/`*` | `\u2733` | 八角星（非 macOS 用 `*`） |
| 3 | `✶` | `\u2736` | 六角实心星 |
| 4 | `✻` | `\u273b` | 泪滴星 |
| 5 | `✽`/`*` | `\u273d` | 花朵星（Ghostty/兼容用 `*`） |

### 2. 帧序列构造

完整帧序列由基础帧正序拼接倒序，形成 12 帧的往返循环，定义在 [`claude-code-source/src/components/Spinner/SpinnerGlyph.tsx`](../../claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L7)：

```typescript
// claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L7
const SPINNER_FRAMES = [...DEFAULT_CHARACTERS, ...[...DEFAULT_CHARACTERS].reverse()];
```

即 `[·, ✢, ✳, ✶, ✻, ✽, ✻, ✶, ✳, ✢, ·]`，实现"从小到大再回到小"的呼吸式动画。帧切换间隔为 120ms，定义在 [`claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx`](../../claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L131)：

```typescript
// claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L131
const frame = reducedMotion ? 0 : Math.floor(time / 120);
```

`SpinnerGlyph` 组件渲染当前帧字符，定义在 [`claude-code-source/src/components/Spinner/SpinnerGlyph.tsx`](../../claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L22-L79)。正常状态下渲染逻辑为：

## 三、 SpinnerMode 状态机

### 1. 模式定义

Spinner 通过 `SpinnerMode` 类型标识当前工作状态。根据源码中 `SpinnerModeGlyph` 的分支（[`claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx`](../../claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L232-L260)），模式包括：

| 模式 | 含义 | 方向图标 |
|---|---|---|
| `requesting` | 正在请求 API（等待首 token） | `↑` 上箭头（`figures.arrowUp`） |
| `thinking` | 模型思考中 | `↓` 下箭头 |
| `responding` | 流式响应中 | `↓` 下箭头 |
| `streaming` | 流式输出 | `↓` 下箭头 |
| `tool-use` | 工具调用中 | `↓` 下箭头 |
| `tool-input` | 等待工具输入 | `↓` 下箭头 |

### 2. SpinnerModeGlyph 图标渲染

模式方向图标由 `SpinnerModeGlyph` 组件根据 `mode` 分派渲染，定义在 [`claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx`](../../claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L232-L264)：

```typescript
// claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L232-L264
function SpinnerModeGlyph({ mode }: { mode: SpinnerMode }) {
  switch (mode) {
    case 'tool-input':
    case 'tool-use':
    case 'responding':
    case 'thinking':
      return (
        <Box width={2}><Text dimColor={true}>{figures.arrowDown}</Text></Box>
      )
    case 'requesting':
      return (
        <Box width={2}><Text dimColor={true}>{figures.arrowUp}</Text></Box>
      )
  }
}
```

所有模式图标均使用 `dimColor` 暗淡渲染，宽度固定为 2 字符。`requesting` 模式显示 `↑`（上箭头，表示请求上传），其余模式显示 `↓`（下箭头，表示接收下载）。

### 3. 模式切换来源

模式状态由 REPL 层维护，通过 `useState` 初始化为 `'responding'`，定义在 [`claude-code-source/src/screens/REPL.tsx`](../../claude-code-source/src/screens/REPL.tsx#L838)：


`setStreamMode` 通过工具上下文（`ToolUseContext`）传递给各调用方，在不同阶段切换模式：

- 发起请求/压缩时设为 `'requesting'`（见 [`claude-code-source/src/services/compact/compact.ts`](../../claude-code-source/src/services/compact/compact.ts#L427)）
- 开始响应时设为 `'responding'`（见 [`claude-code-source/src/services/compact/compact.ts`](../../claude-code-source/src/services/compact/compact.ts#L1340)）
- 工具调用阶段由工具执行钩子切换

思考状态在 `SpinnerWithVerb` 中单独追踪，保证每种状态至少显示 2 秒以避免抖动，定义在 [`claude-code-source/src/components/Spinner.tsx`](../../claude-code-source/src/components/Spinner.tsx#L123-L159)：

- `'thinking'`：正在思考，显示 `thinking` 文字（带微光动画）

- `null`：无思考状态

## 四、 停滞检测与红色渐变

### 1. useStalledAnimation 钩子

当 LLM 停止输出 token 超过 3 秒时，Spinner 字符和消息文字会逐渐变红，提示可能的卡顿。该逻辑由 `useStalledAnimation` 实现，定义在 [`claude-code-source/src/components/Spinner/useStalledAnimation.ts`](../../claude-code-source/src/components/Spinner/useStalledAnimation.ts#L6-L75)：

```typescript
// claude-code-source/src/components/Spinner/useStalledAnimation.ts#L6-L75
export function useStalledAnimation(
  time: number,
  currentResponseLength: number,
  hasActiveTools = false,
  reducedMotion = false,
): { isStalled: boolean; stalledIntensity: number } {
  // 新 token 到达时重置计时器
  if (currentResponseLength > lastResponseLength.current) {
    lastTokenTime.current = time;
    lastResponseLength.current = currentResponseLength;
    stalledIntensityRef.current = 0;
  }
  // 计算距上次 token 的时间
  const isStalled = timeSinceLastToken > 3000 && !hasActiveTools;
  const intensity = isStalled
    ? Math.min((timeSinceLastToken - 3000) / 2000, 1) // 2 秒内渐变到全红
    : 0;
  // 平滑过渡（每 50ms 步进，差值 × 0.1）
}
```

【**函数作用**】检测 LLM 输出是否停滞，并计算停滞强度（0~1），用于驱动 Spinner 向红色渐变。

【**参数含义**】

- `time`：动画时钟当前时间（毫秒）
- `currentResponseLength`：当前响应的字符长度
- `hasActiveTools`：是否有活动工具（为 `true` 时豁免停滞判定）
- `reducedMotion`：是否启用减弱动画

【**返回值**】

返回 `{ isStalled: boolean; stalledIntensity: number }`，`isStalled` 标识是否停滞，`stalledIntensity` 为 0~1 的渐变强度。

### 2. 渐变参数

停滞检测的关键参数：

- **触发阈值**：3000ms 无新 token（且无活动工具）开始变红
- **渐变时长**：从触发到全红用 2000ms（`intensity` 从 0 到 1）
- **平滑步进**：每 50ms 将当前值向目标值靠近 10%（`current += diff * 0.1`），避免突变
- **工具活跃豁免**：`hasActiveTools` 为 `true` 时重置计时器（工具执行期间不算停滞）

### 3. 颜色插值

停滞时，基础色向 `ERROR_RED`（`rgb(171,43,63)`，对应 Light 主题的 `error` 色）插值。`ERROR_RED` 常量定义在 [`claude-code-source/src/components/Spinner/SpinnerGlyph.tsx`](../../claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L10-L14)，停滞插值逻辑在同文件 [`L50-L67`](../../claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L50-L67) 中实现：

```typescript
// claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L10-L14
const ERROR_RED = { r: 171, g: 43, b: 63 };

// L50-L67: 停滞时插值
if (stalledIntensity > 0) {
  const baseRGB = baseColorStr ? parseRGB(baseColorStr) : null;
  if (baseRGB) {
    const interpolated = interpolateColor(baseRGB, ERROR_RED, stalledIntensity);
    return <Text color={toRGBColor(interpolated)}>{spinnerChar}</Text>;
  }
  // ANSI 主题回退：intensity > 0.5 用 error 色
  const color = stalledIntensity > 0.5 ? "error" : messageColor;
}
```

`interpolateColor()` 对 RGB 三通道线性插值，定义在 [`claude-code-source/src/components/Spinner/utils.ts`](../../claude-code-source/src/components/Spinner/utils.ts#L14-L24)：

```typescript
// claude-code-source/src/components/Spinner/utils.ts#L14-L24
export function interpolateColor(color1, color2, t) {
  return {
    r: Math.round(color1.r + (color2.r - color1.r) * t),
    g: Math.round(color1.g + (color2.g - color1.g) * t),
    b: Math.round(color1.b + (color2.b - color1.b) * t),
  }
}
```

## 五、 微光（Glimmer）扫过效果

### 1. GlimmerMessage 组件

`GlimmerMessage` 在 Spinner 消息文字上叠加一道浅色高光，从一端扫向另一端，定义在 [`claude-code-source/src/components/Spinner/GlimmerMessage.tsx`](../../claude-code-source/src/components/Spinner/GlimmerMessage.tsx#L23-L327)。扫过位置由 `glimmerIndex` 决定，在 [`claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx`](../../claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L132-L138) 中计算：

```typescript
// claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L132-L138
const glimmerSpeed = mode === 'requesting' ? 50 : 200;
const cycleLength = glimmerMessageWidth + 20;
const cyclePosition = Math.floor(time / glimmerSpeed);
const glimmerIndex = reducedMotion ? -100
  : isStalled ? -100  // 停滞时禁用微光
  : mode === 'requesting' ? cyclePosition % cycleLength - 10  // 左→右
  : glimmerMessageWidth + 10 - cyclePosition % cycleLength;    // 右→左
```

- `requesting` 模式：微光从左向右扫过（模拟"上传/请求"方向），速度 50ms/步
- 其他模式：微光从右向左扫过（模拟"接收/下载"方向），速度 200ms/步
- 停滞或减弱动画时：`glimmerIndex = -100`（移出文本范围，不显示）

### 2. 三段分割

微光将文本分为三段渲染：`before`（基础色）+ `shim`（浅色高光）+ `after`（基础色），高光窗口为 `glimmerIndex ± 1` 共 3 字符宽。浅色使用 `shimmerColor`（默认 `claudeShimmer`）。

### 3. 工具调用闪烁

`tool-use` 模式下，整个消息以正弦波频率闪烁（基础色↔浅色），定义在 [`claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx`](../../claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L139)：

```typescript
// claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L139
const flashOpacity = reducedMotion ? 0
  : mode === 'tool-use' ? (Math.sin(time / 1000 * Math.PI) + 1) / 2
  : 0;
```

`flashOpacity` 在 0~1 间正弦波动（周期 2 秒），通过 `interpolateColor` 在 `messageColor` 和 `shimmerColor` 间插值。

## 七、 减弱动画模式

### 1. Reduced Motion 处理

当 `prefersReducedMotion` 为 `true` 时，所有动画降级为静态或缓变：

- Spinner 字符固定为第 0 帧（`frame = 0`）
- 微光禁用（`glimmerIndex = -100`）
- 工具闪烁禁用（`flashOpacity = 0`）
- 停滞强度即时跳变（不做平滑过渡）

### 2. 静态圆点替代

减弱动画下，Spinner 字符替换为 `●`（实心圆），以 2 秒周期明暗闪烁。常量定义在 [`claude-code-source/src/components/Spinner/SpinnerGlyph.tsx`](../../claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L8-L9)，渲染逻辑在同文件 [`L36-L47`](../../claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L36-L47)：

```typescript
// claude-code-source/src/components/Spinner/SpinnerGlyph.tsx#L8-L9
const REDUCED_MOTION_DOT = '●';
const REDUCED_MOTION_CYCLE_MS = 2000; // 1s 可见，1s 暗

// L36-L47
if (reducedMotion) {
  const isDim = Math.floor(time / (REDUCED_MOTION_CYCLE_MS / 2)) % 2 === 1;
  return <Text color={messageColor} dimColor={isDim}>{REDUCED_MOTION_DOT}</Text>;
}
```

## 十、 动画时钟架构

### 1. 50ms 主时钟

`SpinnerAnimationRow` 通过 `useAnimationFrame(50)` 订阅 50ms 动画时钟，定义在 [`claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx`](../../claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L103)：

```typescript
// claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L103
const [viewportRef, time] = useAnimationFrame(reducedMotion ? null : 50);
```

所有动画推导（帧序、微光位置、停滞强度、token 计数动画、计时器）都从共享的 `time` 派生，避免多个独立定时器。父组件 `SpinnerWithVerb` 不在 50ms 循环中，仅在 props/状态变化时重渲染（每轮约 25 次），将外层 Box、状态选择器等排除在热路径外。

### 2. Token 计数平滑

Token 计数不直接显示真实值，而是以平滑增量追赶，定义在 [`claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx`](../../claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L142-L159)：

```typescript
// claude-code-source/src/components/Spinner/SpinnerAnimationRow.tsx#L142-L158
const gap = currentResponseLength - tokenCounterRef.current;
if (gap > 0) {
  let increment;
  if (gap < 70) {
    increment = 3;
  } else if (gap < 200) {
    increment = Math.max(8, Math.ceil(gap * 0.15));
  } else {
    increment = 50;
  }
  tokenCounterRef.current = Math.min(tokenCounterRef.current + increment, currentResponseLength);
}
```

增量分三档：小差距（<70）每帧 +3，中差距（<200）按 15% 追赶，大差距（≥200）每帧 +50，使数字滚动既流畅又及时。
