# 2026-04-01

## 添加 OpenAI 兼容 API 支持

为 claude-code-source 添加了 OpenAI 兼容 API 支持，通过 fetch 层适配器做 Anthropic ↔ OpenAI 协议双向转换。

### 修改的文件

1. **新增** `src/services/api/openai-adapter.ts` — 核心适配层
   - `createOpenAIAdapterFetch()` — 拦截 Anthropic SDK 的 fetch 请求
   - `convertAnthropicRequestToOpenAI()` — 请求格式转换（messages、tools、system prompt）
   - `createOpenAIToAnthropicStreamTransformer()` — SSE 流式响应转换
   - `convertOpenAINonStreamingToAnthropic()` — 非流式响应转换

2. **修改** `src/utils/model/providers.ts`
   - `APIProvider` 类型加了 `'openai'`
   - `getAPIProvider()` 新增 `CLAUDE_CODE_USE_OPENAI` 检测（最高优先级）
   - `isFirstPartyAnthropicBaseUrl()` 在 OpenAI 模式下返回 false

3. **修改** `src/services/api/client.ts`
   - `getAnthropicClient()` 新增 OpenAI 分支：创建带适配器 fetch 的 Anthropic 客户端

4. **修改** `src/utils/model/configs.ts`
   - 所有 ModelConfig 添加 `openai` 字段

5. **修改** `src/utils/auth.ts`
   - `isAnthropicAuthEnabled()` 中的 `is3P` 条件添加 OpenAI

6. **修改** `src/utils/managedEnvConstants.ts`
   - 注册 `CLAUDE_CODE_USE_OPENAI`、`OPENAI_BASE_URL`、`OPENAI_API_KEY`、`OPENAI_MODEL`

7. **修改** `README.md` — 添加完整的 OpenAI 兼容 API 使用文档

### 环境变量

- `CLAUDE_CODE_USE_OPENAI=1` — 启用 OpenAI 兼容模式
- `OPENAI_API_KEY` — API Key
- `OPENAI_BASE_URL` — API 基础 URL（需包含 /v1）
- `OPENAI_MODEL` — 模型名称（可选）
