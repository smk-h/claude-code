# 2026-04-01

## 添加 OpenAI 兼容 API 支持

为 claude-code-source 添加了 OpenAI 兼容 API 支持，通过 fetch 层适配器做 Anthropic ↔ OpenAI 协议双向转换。

### 修改的文件

1. **新增** `src/services/api/openai-adapter.ts` — 核心适配层
   - `createOpenAIAdapterFetch()` — 拦截 Anthropic SDK 的 fetch 请求
   - `convertAnthropicRequestToOpenAI()` — 请求格式转换（messages、tools、system prompt）
   - `createOpenAIToAnthropicStreamTransformer()` — SSE 流式响应转换
   - `convertOpenAINonStreamingToAnthropic()` — 非流式响应转换

2. **修改** `src/utils/model/providers.ts`
   - `APIProvider` 类型加了 `'openai'`
   - `getAPIProvider()` 新增 `CLAUDE_CODE_USE_OPENAI` 检测（最高优先级）
   - `isFirstPartyAnthropicBaseUrl()` 在 OpenAI 模式下返回 false

3. **修改** `src/services/api/client.ts`
   - `getAnthropicClient()` 新增 OpenAI 分支：创建带适配器 fetch 的 Anthropic 客户端

4. **修改** `src/utils/model/configs.ts`
   - 所有 ModelConfig 添加 `openai` 字段

5. **修改** `src/utils/auth.ts`
   - `isAnthropicAuthEnabled()` 中的 `is3P` 条件添加 OpenAI

6. **修改** `src/utils/managedEnvConstants.ts`
   - 注册 `CLAUDE_CODE_USE_OPENAI`、`OPENAI_BASE_URL`、`OPENAI_API_KEY`、`OPENAI_MODEL`

7. **修改** `README.md` — 添加完整的 OpenAI 兼容 API 使用文档

### 环境变量

- `CLAUDE_CODE_USE_OPENAI=1` — 启用 OpenAI 兼容模式
- `OPENAI_API_KEY` — API Key
- `OPENAI_BASE_URL` — API 基础 URL（需包含 /v1）
- `OPENAI_MODEL` — 模型名称（可选）

## 补漏修复：确保 OpenAI 不影响原有逻辑

全面审查后发现多处遗漏，做了以下补充修改：

1. **`src/utils/auth.ts`** — `isUsing3PServices()` 和 `is1PApiCustomer()` 都添加了 `CLAUDE_CODE_USE_OPENAI` 检查
2. **`src/entrypoints/sdk/coreSchemas.ts`** — `AccountInfoSchema` 的 `apiProvider` z.enum 添加 `'openai'`
3. **`src/utils/managedEnvConstants.ts`** — `SAFE_ENV_VARS` set 添加 `'CLAUDE_CODE_USE_OPENAI'`
4. **`src/services/analytics/config.ts`** — `isAnalyticsDisabled()` 添加 OpenAI 检查
5. **`src/commands/feedback/index.ts`** — feedback 命令在 OpenAI 模式下禁用
6. **`src/utils/apiPreconnect.ts`** — OpenAI 模式跳过 Anthropic API 预连接
7. **`src/utils/log.ts`** — OpenAI 模式禁用 error reporting
8. **`src/utils/swarm/spawnUtils.ts`** — TEAMMATE_ENV_VARS 添加 OpenAI 相关环境变量转发

### 确认不需要修改的地方（原有逻辑已兼容）

- `!== 'firstParty'` 检查：OpenAI 自动被排除，语义正确
- `betas.ts`：`shouldIncludeFirstPartyOnlyBetas` 只在 firstParty/foundry 返回 true，OpenAI 正确被排除
- `thinking.ts`：OpenAI 不会匹配 firstParty/foundry，不会启用 extended thinking
- `WebSearchTool`：OpenAI provider 走到 default 分支返回 false，web search 禁用（合理）
- `tokenEstimation.ts`：countTokens 在 bedrock 时有特殊处理，OpenAI 走默认路径（OK）
- `buildFetch` 中的 `injectClientRequestId`：仅在 firstParty + isFirstPartyAnthropicBaseUrl() 时注入，OpenAI 不会匹配

## Dockerfile 移到根目录

CNB 构建报错 `open Dockerfile: no such file or directory`，因为 Dockerfile 原在 `claude-code-source/` 子目录下，而 `.cnb.yml` 的 docker build 在根目录执行。将 Dockerfile 移到根目录，所有 COPY 路径加了 `claude-code-source/` 前缀。同时解决了 Git 合并冲突。
