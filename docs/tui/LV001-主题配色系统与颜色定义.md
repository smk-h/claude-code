## 一、 概述

Claude Code 的命令行界面（TUI）基于 React + Ink 构建，采用了一套结构化的主题配色系统。所有颜色定义集中在一个主题文件中，通过 `Theme` 接口约束约 75 个颜色槽位，并为 6 套内置主题分别提供具体的 RGB 或 ANSI 取值。本文全面解析主题系统的类型定义、各主题的取值，以及每个颜色槽位的用途。

主题系统的核心源码位于 [`claude-code-source/src/utils/theme.ts`](../../claude-code-source/src/utils/theme.ts)。

## 二、 主题类型定义

### 2. 完整颜色槽位分类

Theme 接口的颜色槽位按功能可分为以下几组：

#### 2.1 品牌与核心色

- `claude`：Claude 品牌橙色，用于欢迎页标题、助手消息圆点、加载图标默认色
- `claudeShimmer`：品牌橙的浅色变体，用于微光（shimmer）动画效果
- `claudeBlue_FOR_SYSTEM_SPINNER`：系统级加载图标的蓝色
- `claudeBlueShimmer_FOR_SYSTEM_SPINNER`：系统加载图标蓝色的浅色变体

#### 2.2 状态与模式色

- `permission`：权限请求提示色（蓝紫色调）
- `permissionShimmer`：权限色的浅色变体
- `planMode`：计划模式色（灰绿色调）
- `ide`：IDE 连接色（蓝色调）
- `promptBorder`：输入框边框色（灰色调）
- `promptBorderShimmer`：输入框边框浅色变体
- `autoAccept`：自动接受模式色（电紫色）
- `bashBorder`：Bash 工具边框色（粉色/蓝色调）
- `fastMode`：快速模式色（电橙色）
- `fastModeShimmer`：快速模式浅色变体

#### 2.3 文本色

- `text`：主要文本色（深色主题为白色，浅色主题为黑色）
- `inverseText`：反色文本（与 `text` 相反）
- `inactive`：非活动/灰色文本
- `inactiveShimmer`：非活动文本浅色变体
- `subtle`：暗淡文本（用于次要信息）
- `suggestion`：建议文本色（蓝紫色调）
- `remember`：记忆提示色
- `background`：通用背景色（深色主题为 `rgb(0,204,204)` 亮青色，用于 asciichart 图表等场景）

#### 2.4 语义色

- `success`：成功状态（绿色）
- `error`：错误状态（红色）
- `warning`：警告状态（琥珀色）
- `warningShimmer`：警告浅色变体
- `merged`：合并/分支色（电紫色，与 `autoAccept` 一致）

#### 2.5 Diff 差异色

- `diffAdded`：新增行背景色
- `diffRemoved`：删除行背景色
- `diffAddedDimmed`：新增行暗色背景
- `diffRemovedDimmed`：删除行暗色背景
- `diffAddedWord`：单词级新增高亮
- `diffRemovedWord`：单词级删除高亮

#### 2.6 子 Agent 颜色（8 种）

专用于子 Agent 标识的 8 种颜色：`red_FOR_SUBAGENTS_ONLY`、`blue_FOR_SUBAGENTS_ONLY`、`green_FOR_SUBAGENTS_ONLY`、`yellow_FOR_SUBAGENTS_ONLY`、`purple_FOR_SUBAGENTS_ONLY`、`orange_FOR_SUBAGENTS_ONLY`、`pink_FOR_SUBAGENTS_ONLY`、`cyan_FOR_SUBAGENTS_ONLY`。

#### 2.7 TUI V2 界面色

- `clawd_body`：Claude 吉祥物主体色
- `clawd_background`：吉祥物背景色
- `userMessageBackground`：用户消息背景色
- `userMessageBackgroundHover`：用户消息悬停背景色
- `messageActionsBackground`：消息操作选中背景色（带冷色调蓝色偏移）
- `selectionBg`：文本选择高亮背景色
- `bashMessageBackgroundColor`：Bash 消息背景色
- `memoryBackgroundColor`：记忆消息背景色
- `rate_limit_fill`：速率限制条填充色
- `rate_limit_empty`：速率限制条空色
- `briefLabelYou`：Brief 模式 "You" 标签色
- `briefLabelClaude`：Brief 模式 "Claude" 标签色
- `professionalBlue`：专业蓝色
- `chromeYellow`：Chrome 黄色

#### 2.8 彩虹色（ultrathink 关键词高亮）

7 种彩虹色及其浅色变体，用于 `ultrathink` 关键词的彩虹高亮：`rainbow_red`、`rainbow_orange`、`rainbow_yellow`、`rainbow_green`、`rainbow_blue`、`rainbow_indigo`、`rainbow_violet`，以及对应的 `_shimmer` 变体。

## 三、 主题名称与解析


### 2. 主题解析函数

`getTheme()` 函数根据主题名返回具体的 `Theme` 对象，默认返回 `darkTheme`：

```typescript
// claude-code-source/src/utils/theme.ts#L598-L613
export function getTheme(themeName: ThemeName): Theme {
  switch (themeName) {
    case 'light':
      return lightTheme
    case 'light-ansi':
      return lightAnsiTheme
    case 'dark-ansi':
      return darkAnsiTheme
    case 'light-daltonized':
      return lightDaltonizedTheme
    case 'dark-daltonized':
      return darkDaltonizedTheme
    default:
      return darkTheme
  }
}
```

- `claude` 改为 `rgb(255,153,51)`（针对绿色盲调整的橙色）

### 4. ANSI 主题（16 色终端兼容）

`light-ansi` 与 `dark-ansi` 主题仅使用 16 种标准 ANSI 颜色，用于不支持真彩色的终端。定义在 [`claude-code-source/src/utils/theme.ts`](../../claude-code-source/src/utils/theme.ts#L197-L272) 和 [`claude-code-source/src/utils/theme.ts`](../../claude-code-source/src/utils/theme.ts#L278-L353)。颜色值使用 `ansi:` 前缀格式，例如：

- `claude: 'ansi:redBright'`（亮红替代品牌橙）
- `text: 'ansi:black'`（浅色）/ `'ansi:whiteBright'`（深色）
- `success: 'ansi:green'`（浅色）/ `'ansi:greenBright'`（深色）
- `error: 'ansi:red'`（浅色）/ `'ansi:redBright'`（深色）
- `userMessageBackground: 'ansi:white'`（浅色）/ `'ansi:blackBright'`（深色）

## 五、 颜色工具函数

### 1. themeColorToAnsi()

该函数将主题颜色转换为 ANSI 转义序列，供 asciichart 图表库使用。针对 Apple Terminal 使用 256 色模式，定义在 [`claude-code-source/src/utils/theme.ts`](../../claude-code-source/src/utils/theme.ts#L615-L639)：

```typescript
// claude-code-source/src/utils/theme.ts#L615-L639
const chalkForChart =
  env.terminal === 'Apple_Terminal'
    ? new Chalk({ level: 2 }) // 256 colors
    : chalk

export function themeColorToAnsi(themeColor: string): string {
  const rgbMatch = themeColor.match(/rgb\(\s?(\d+),\s?(\d+),\s?(\d+)\s?\)/)
  if (rgbMatch) {
    const r = parseInt(rgbMatch[1]!, 10)
    const g = parseInt(rgbMatch[2]!, 10)
    const b = parseInt(rgbMatch[3]!, 10)
    const colored = chalkForChart.rgb(r, g, b)('X')
    return colored.slice(0, colored.indexOf('X'))
  }
  return '\x1b[35m' // 解析失败回退到 magenta
}
```

【**函数作用**】

将 `rgb(r,g,b)` 格式的主题色字符串转换为终端 ANSI 转义序列，供不支持直接 RGB 的图表库使用。

【**参数含义**】

- `themeColor`：主题色字符串，格式如 `rgb(215,119,87)` 或 `ansi:redBright`

【**返回值**】

返回 ANSI 转义序列字符串；解析失败时回退到 `\x1b[35m`（magenta）。

### 2. Ink 中的颜色解析

在 Ink 组件中，`<Text color="claude">` 这样的写法会被 Ink 内部解析：若值为 `Theme` 接口的键名，则通过 `getTheme()` 查找实际 RGB/ANSI 值再渲染。这使得组件代码可直接引用语义化颜色名（如 `claude`、`error`、`subtle`），而无需关心具体取值。

## 六、 主题设计要点

### 1. Shimmer（微光）机制

多数颜色槽位成对出现：一个基础色和一个 `Shimmer` 变体。Shimmer 变体是基础色的浅色版本，用于动画中的高亮扫过效果。例如 `claude`（`rgb(215,119,87)`）与 `claudeShimmer`（`rgb(235,159,127)`），后者通过提升 RGB 各通道值实现"更亮"的视觉感。

### 2. 对比度量化

用户消息背景色的取值经过量化设计，确保在 256 色终端下仍可区分。例如 Light 主题中 `userMessageBackground=rgb(240,240,240)` 与 `userMessageBackgroundHover=rgb(252,252,252)` 的差值经过计算，保证在 256 色量化后不坍缩为同一色。

### 3. 色盲友好策略

Daltonized 主题将红绿区分替换为蓝黄区分：绿色盲（deuteranopia）难以区分红绿，因此将 `success`（成功绿）改为蓝色，`diffAdded`（新增绿）改为浅蓝，`bashBorder`（粉色）改为蓝色，保证关键状态在色盲用户眼中仍可辨识。
