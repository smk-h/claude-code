## 一、 总体架构

Claude Code 的终端光标系统由两层协同构成：

- **视觉光标层**：通过 `chalk.inverse()` 渲染反色字符，在终端输出中"画"出一个静态块状光标，用户可见
- **物理光标层**：通过 `useDeclaredCursor` Hook 将终端原生光标停放（park）到输入位置，终端原生光标被隐藏（`CSI ?25l`），但对 IME 输入法和屏幕阅读器仍可追踪

两层光标各司其职：视觉光标负责展示，物理光标负责输入法与无障碍支持。

### 1. 数据流总览

```
用户按键 → useTextInput → Cursor.render(invert) → 渲染反色光标字符
                        → useDeclaredCursor → CursorDeclaration → ink.tsx → 终端光标停放
```

## 二、 视觉光标：invert 反色机制

### 1. invert 函数的三种模式

`invert` 函数定义在 [`src/components/TextInput.tsx`](../../claude-code-source/src/components/TextInput.tsx#L65-L91) 中，根据运行状态分为三种模式：

```typescript
// src/components/TextInput.tsx:65-91
const canShowCursor = isTerminalFocused && !accessibilityEnabled;
let invert: (text: string) => string;

if (!canShowCursor) {
  // 模式一：终端失焦或无障碍模式 — 不显示光标
  invert = (text: string) => text;
} else if (isVoiceRecording && !reducedMotion) {
  // 模式二：语音录制 — 波形动画光标
  invert = () => chalk.rgb(r, g, b)(BARS[barIndex]!);
} else {
  // 模式三：标准输入 — 反色块状光标
  invert = chalk.inverse;
}
```

#### 1.1 模式一：隐藏光标

当终端窗口失去焦点（`!isTerminalFocused`）或开启了无障碍模式（`CLAUDE_CODE_ACCESSIBILITY` 环境变量）时，`invert` 为恒等函数，光标不显示。此时依赖操作系统原生的光标闪烁来指示输入位置。

#### 1.2 模式二：语音录制波形光标

语音录制时，光标变为动态波形：

- 使用 Unicode 块元素字符 `▁▂▃▄▅▆▇█` 表示音量高度
- 静音时显示灰色，说话时颜色在色相环上循环
- 采用指数移动平均（EMA，平滑系数 0.7）消除抖动
- 动画帧率 50ms（通过 `useAnimationFrame` 驱动）

相关常量定义在 [`src/components/TextInput.tsx`](../../claude-code-source/src/components/TextInput.tsx#L16-L33)：

```typescript
// src/components/TextInput.tsx:16-33
const BARS = ' \u2581\u2582\u2583\u2584\u2585\u2586\u2587\u2588';
const CURSOR_WAVEFORM_WIDTH = 1;
const SMOOTH = 0.7;
const LEVEL_BOOST = 1.8;
const SILENCE_THRESHOLD = 0.15;
```

#### 1.3 模式三：标准反色光标

日常输入状态下使用 `chalk.inverse`，即 ANSI SGR 7（Reverse Video）转义序列 `\x1b[7m...\x1b[27m`，将光标位置字符的前景色与背景色互换。对于空格字符 `invert(' ')`，效果为一个实心白色方块。

### 2. cursorChar 的传递

`cursorChar` 决定光标位置渲染哪个字符，定义在 [`src/components/TextInput.tsx`](../../claude-code-source/src/components/TextInput.tsx#L105)：

```typescript
// src/components/TextInput.tsx:105
cursorChar: props.showCursor ? ' ' : '',
```

- `showCursor = true` → `cursorChar = ' '`（空格），经 `invert(' ')` 后变为反色实心块
- `showCursor = false` → `cursorChar = ''`（空字符串），`invert('')` 仍为空，光标消失

### 3. Cursor.render() 渲染逻辑

`Cursor` 类的 `render()` 方法负责将文本和光标合成为最终输出，定义在 [`src/utils/Cursor.ts`](../../claude-code-source/src/utils/Cursor.ts#L203-L299)：

```typescript
// src/utils/Cursor.ts:203-299
render(
  cursorChar: string,
  mask: string,
  invert: (text: string) => string,
  ghostText?: { text: string; dim: (text: string) => string },
  maxVisibleLines?: number,
) {
  // ...逐行遍历...

  // 在光标所在行，将文本分为 before / at / after 三段
  let beforeCursor = ''
  let atCursor = cursorChar       // 默认：光标位置显示 cursorChar
  let afterCursor = ''

  // 逐 grapheme 遍历，累积显示宽度直到到达光标列
  for (const { segment } of getGraphemeSegmenter().segment(displayText)) {
    if (cursorFound) {
      afterCursor += segment
      continue
    }
    const nextWidth = currentWidth + stringWidth(segment)
    if (nextWidth > column) {
      atCursor = segment            // 光标落在某个字符上时，用该字符替代空格
      cursorFound = true
    } else {
      currentWidth = nextWidth
      beforeCursor += segment
    }
  }

  // 仅当 cursorChar 非空时才对光标字符应用 invert
  renderedCursor = cursorChar ? invert(atCursor) : atCursor

  return beforeCursor + renderedCursor + ghostSuffix + afterCursor.trimEnd()
}
```

关键细节：

- 光标在行尾时，`atCursor = cursorChar = ' '`，`invert(' ')` 产生实心块
- 光标在某个字符上时，`atCursor = segment`（该字符），`invert(segment)` 使该字符反色显示
- Ghost Text（内联补全建议）存在时，第一个建议字符放在反色光标中，剩余部分 dim 显示

## 三、 占位符光标：renderPlaceholder

当输入为空时，光标需要渲染在 placeholder 文本上。此逻辑由 [`src/hooks/renderPlaceholder.ts`](../../claude-code-source/src/hooks/renderPlaceholder.ts#L13-L51) 处理：

```typescript
// src/hooks/renderPlaceholder.ts:13-51
export function renderPlaceholder({
  placeholder,
  value,
  showCursor,
  focus,
  terminalFocus = true,
  invert = chalk.inverse,
  hidePlaceholderText = false,
}): { renderedPlaceholder: string | undefined; showPlaceholder: boolean } {
  let renderedPlaceholder: string | undefined = undefined

  if (placeholder) {
    if (hidePlaceholderText) {
      // 语音录制模式：只显示光标，不显示占位文本
      renderedPlaceholder =
        showCursor && focus && terminalFocus ? invert(' ') : ''
    } else {
      renderedPlaceholder = chalk.dim(placeholder)

      // 仅在输入框和终端均获得焦点时显示反色光标
      if (showCursor && focus && terminalFocus) {
        renderedPlaceholder =
          placeholder.length > 0
            ? invert(placeholder[0]!) + chalk.dim(placeholder.slice(1))
            : invert(' ')
      }
    }
  }

  const showPlaceholder = value.length === 0 && Boolean(placeholder)

  return { renderedPlaceholder, showPlaceholder }
}
```

### 1. 渲染策略

| 条件 | 渲染结果 |
|---|---|
| 空输入 + 有 placeholder + 聚焦 | `invert(placeholder[0])` + `dim(剩余字符)` — 第一个字符反色，其余暗淡 |
| 空输入 + 有 placeholder + 失焦 | `dim(placeholder)` — 整体暗淡，无光标 |
| 空输入 + 空 placeholder + 聚焦 | `invert(' ')` — 实心块光标 |
| 语音录制模式 | `invert(' ')` — 仅块状光标，不显示占位文本 |

### 2. 三重条件控制

光标仅在以下三个条件同时满足时才显示：

- `showCursor` — 组件配置允许显示光标
- `focus` — 输入框获得焦点
- `terminalFocus` — 终端窗口本身获得焦点

这个三重条件确保了切换窗口时光标正确消失，避免残留的视觉光标误导用户。

## 四、 物理光标：useDeclaredCursor

### 1. Hook 设计

[`useDeclaredCursor`](../../claude-code-source/src/ink/hooks/use-declared-cursor.ts#L25-L73) 将终端原生光标停放（park）到输入插入符位置，定义如下：

```typescript
// src/ink/hooks/use-declared-cursor.ts:25-73
export function useDeclaredCursor({
  line,
  column,
  active,
}: {
  line: number
  column: number
  active: boolean
}): (element: DOMElement | null) => void {
  const setCursorDeclaration = useContext(CursorDeclarationContext)
  const nodeRef = useRef<DOMElement | null>(null)

  const setNode = useCallback((node: DOMElement | null) => {
    nodeRef.current = node
  }, [])

  // 每个 commit 重新声明位置（无 dep array）
  useLayoutEffect(() => {
    const node = nodeRef.current
    if (active && node) {
      setCursorDeclaration({ relativeX: column, relativeY: line, node })
    } else {
      setCursorDeclaration(null, node)
    }
  })

  // 卸载时条件清除
  useLayoutEffect(() => {
    return () => {
      setCursorDeclaration(null, nodeRef.current)
    }
  }, [setCursorDeclaration])

  return setNode
}
```

### 2. 调用位置

在 [`BaseTextInput`](../../claude-code-source/src/components/BaseTextInput.tsx#L38-L53) 中被调用：

```typescript
// src/components/BaseTextInput.tsx:38-53
const t1 = Boolean(props.focus && props.showCursor && terminalFocus);

const cursorRef = useDeclaredCursor({
  line: cursorLine,
  column: cursorColumn,
  active: t1,   // 同样的三重条件：focus && showCursor && terminalFocus
});
```

### 3. 节点身份校验

`setCursorDeclaration` 的第二个参数 `clearIfNode` 用于条件清除。在 [`ink.tsx`](../../claude-code-source/src/ink/ink.tsx#L1436-L1441) 中的实现：

```typescript
// src/ink/ink.tsx:1436-1441
private setCursorDeclaration: CursorDeclarationSetter = (decl, clearIfNode) => {
  if (decl === null && clearIfNode !== undefined && this.cursorDeclaration?.node !== clearIfNode) {
    return;   // 不清除：当前声明属于其他节点
  }
  this.cursorDeclaration = decl;
};
```

这解决了两个场景的竞争问题：

- **memo 组件不重渲染**：一个被 `memo()` 包裹的活跃输入组件不会因其他实例的状态变更而重渲染，非活跃实例的清除操作不会错误覆盖
- **兄弟组件焦点切换**：焦点从列表项 A 移到 B 时，A 的 `useLayoutEffect` 可能在 B 之后执行，节点身份校验防止 A 的清除操作覆盖 B 的新声明

### 4. 从声明到终端光标移动

在 [`ink.tsx`](../../claude-code-source/src/ink/ink.tsx#L659-L734) 中，每帧渲染后将光标声明转换为终端 CSI 指令：

```typescript
// src/ink/ink.tsx:659-734
const decl = this.cursorDeclaration;
const rect = decl !== null ? nodeCache.get(decl.node) : undefined;
const target = decl !== null && rect !== undefined ? {
  x: rect.x + decl.relativeX,
  y: rect.y + decl.relativeY
} : null;

if (target !== null) {
  if (this.altScreenActive) {
    // Alt Screen：绝对定位 CSI row;col H
    const row = Math.min(Math.max(target.y + 1, 1), terminalRows);
    const col = Math.min(Math.max(target.x + 1, 1), terminalWidth);
    optimized.push({ type: 'stdout', content: cursorPosition(row, col) });
  } else {
    // Main Screen：相对移动
    const from = !hasDiff && parked !== null ? parked : { x: frame.cursor.x, y: frame.cursor.y };
    const dx = target.x - from.x;
    const dy = target.y - from.y;
    if (dx !== 0 || dy !== 0) {
      optimized.push({ type: 'stdout', content: cursorMove(dx, dy) });
    }
  }
  this.displayCursor = target;
}
```

- **Alt Screen**：使用绝对定位 `CSI row;col H`
- **Main Screen**：使用相对移动，从上一帧的光标位置计算偏移量，减少输出字节数

## 五、 原生终端光标的隐藏

### 1. 挂载时隐藏

- 无障碍模式：保留原生光标可见，供屏幕放大器等工具追踪

## 六、 光标不闪烁的设计决策

Claude Code 的输入光标是**静态反色块**，不闪烁。这是有意为之的设计：

1. **原生光标已隐藏**：`CSI ?25l` 让终端不再显示闪烁光标
2. **视觉光标是"画"出来的**：`chalk.inverse(' ')` 渲染为静态反色空格，随 Ink 框架的 diff-render 机制更新，无任何 `setInterval` 或动画帧驱动
3. **`useBlink` Hook 未用于输入**：`useBlink` 仅用于加载 Spinner 等 UI 元素，不用于输入光标
4. **唯一的动态光标是语音录制波形**：此时由 `useAnimationFrame(50ms)` 驱动，属于特殊交互模式

## 七、 完整渲染流程

以用户在输入框中键入一个字符为例，完整的数据流如下：

```
1. 终端发送按键事件
2. useInput 回调触发 onInput
3. useTextInput 更新 value + cursorOffset
4. React 重渲染 TextInput
5. TextInput 计算 invert 函数（根据终端焦点、语音状态等）
6. Cursor.render(cursorChar=' ', mask='', invert) 执行
   → 在光标位置渲染 invert(' ')（实心反色块）
7. BaseTextInput 调用 renderPlaceholder（空输入时）
   → invert(placeholder[0]) + dim(placeholder.slice(1))
8. useDeclaredCursor 在 useLayoutEffect 中声明新的光标位置
9. ink.tsx onRender 读取 cursorDeclaration
   → 计算 nodeCache 中的绝对坐标
   → 发出 cursorMove(dx, dy) 指令
   → 更新 displayCursor
10. 终端接收 diff + 光标移动指令，完成帧渲染
```
