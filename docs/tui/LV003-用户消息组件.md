## 一、 概述

用户消息组件负责在终端界面中渲染用户输入的文本及其衍生内容（如 Bash 输出、命令消息、附件等）。Claude Code 采用分层组件架构：从消息分发器到具体消息渲染器，每一层承担不同的职责。本文深入分析用户消息组件的渲染管线、各子组件的结构与样式，以及 Brief 布局模式的特殊处理。

用户消息组件的核心源码位于 [`claude-code-source/src/components/messages/`](../../claude-code-source/src/components/messages/) 目录。

## 二、 消息分发架构

### 1. Message 分发器

`Message` 组件是所有消息渲染的入口，根据消息类型分派到不同的子组件。该组件在 [`claude-code-source/src/components/Message.tsx`](../../claude-code-source/src/components/Message.tsx) 中定义，其分派逻辑处理以下消息类型：

- `case "attachment"` → 渲染 `<AttachmentMessage>`（附件消息）
- `case "assistant"` → 将内容映射到 `<AssistantMessageBlock>`（助手消息块）
- `case "user"` → 将内容映射到 `<UserMessage>`（用户消息，进一步分发）
- `case "system"` → 渲染系统消息、压缩边界等

### 2. UserTextMessage 路由器

`UserTextMessage` 是用户文本的路由组件，根据文本内容中的特殊标签（如 `<bash-stdout>`、`<command-message>`）分派到专门子组件。该组件在 [`claude-code-source/src/components/messages/UserTextMessage.tsx`](../../claude-code-source/src/components/messages/UserTextMessage.tsx) 中定义，分发规则如下：

- `<bash-stdout>` / `<bash-stderr>` → `<UserBashOutputMessage>`
- `<local-command-stdout>` / `<local-command-stderr>` → `<UserLocalCommandOutputMessage>`
- `<bash-input>` → `<UserBashInputMessage>`
- `<command-message>` → `<UserCommandMessage>`
- `<user-memory-input>` → `<UserMemoryInputMessage>`
- `<teammate-message>` → `<UserTeammateMessage>`
- `<task-notification>` → `<UserAgentNotificationMessage>`
- `<mcp-resource-update>` → `<UserResourceUpdateMessage>`
- `<channel source="` → `<UserChannelMessage>`
- 中断消息 → `<InterruptedByUser>`
- 默认 → `<UserPromptMessage>`

## 三、 UserPromptMessage 主组件

### 1. 组件定义与职责

`UserPromptMessage` 是渲染用户实际输入文本的核心组件，负责文本截断、背景色选择和布局控制。该组件在 [`claude-code-source/src/components/messages/UserPromptMessage.tsx`](../../claude-code-source/src/components/messages/UserPromptMessage.tsx#L31-L79) 中定义：

```typescript
// claude-code-source/src/components/messages/UserPromptMessage.tsx#L31-L79
export function UserPromptMessage({
  addMargin,
  param: { text },
  isTranscriptMode,
  timestamp
}: Props): React.ReactNode {
  // ... Brief 布局判断与截断逻辑 ...
  const isSelected = useContext(MessageActionsSelectedContext);
  if (!text) {
    logError(new Error('No content found in user prompt message'));
    return null;
  }
  return <Box flexDirection="column" marginTop={addMargin ? 1 : 0}
      backgroundColor={isSelected ? 'messageActionsBackground'
        : useBriefLayout ? undefined : 'userMessageBackground'}
      paddingRight={useBriefLayout ? 0 : 1}>
      <HighlightedThinkingText text={displayText}
        useBriefLayout={useBriefLayout}
        timestamp={useBriefLayout ? timestamp : undefined} />
    </Box>;
}
```

### 2. 文本截断策略

为避免超大输入（如 `cat 11k-line-file | claude`）导致全屏渲染器性能下降，组件对超长文本实施头尾截断。常量定义在 [`claude-code-source/src/components/messages/UserPromptMessage.tsx`](../../claude-code-source/src/components/messages/UserPromptMessage.tsx#L28-L30)：

```typescript
// claude-code-source/src/components/messages/UserPromptMessage.tsx#L28-L30
const MAX_DISPLAY_CHARS = 10_000;
const TRUNCATE_HEAD_CHARS = 2_500;
const TRUNCATE_TAIL_CHARS = 2_500;
```

截断逻辑保留头部 2500 字符和尾部 2500 字符，中间用 `… +N lines …` 提示省略的行数。采用头尾而非仅头部的原因是：`{ cat file; echo prompt; } | claude` 这类用法会将用户的实际问题放在末尾，尾部必须保留。

### 3. 背景色选择逻辑

背景色根据上下文三态切换：

- 选中状态（`isSelected`）：使用 `messageActionsBackground`（冷灰偏蓝色调，与默认背景区分）
- Brief 布局：不设背景色（`undefined`），因 Brief 模式采用标签式布局
- 默认：使用 `userMessageBackground`（深色主题为 `rgb(55,55,55)` 浅灰）

`paddingRight={1}` 为右侧留出 1 字符内边距（Brief 模式下为 0），使文本与背景边缘有呼吸感。

## 四、 HighlightedThinkingText 文本渲染

### 1. 组件职责

`HighlightedThinkingText` 负责用户文本的实际字符级渲染，包括 `ultrathink` 关键词的彩虹高亮和选中状态的指针颜色切换。该组件在 [`claude-code-source/src/components/messages/HighlightedThinkingText.tsx`](../../claude-code-source/src/components/messages/HighlightedThinkingText.tsx#L15-L161) 中定义。

### 2. 指针前缀与颜色

普通模式下，用户文本以 `figures.pointer`（`›` 指针符号）作为前缀，颜色根据选中状态切换：

```typescript
// claude-code-source/src/components/messages/HighlightedThinkingText.tsx#L24
const pointerColor = isSelected ? "suggestion" : "subtle";
```

- 未选中：`subtle`（深色主题为 `rgb(80,80,80)` 深灰）
- 选中：`suggestion`（深色主题为 `rgb(177,185,249)` 浅蓝紫）

正文文本使用 `text` 颜色（深色主题为白色）。

### 3. ultrathink 彩虹高亮

当用户输入包含 `ultrathink` 关键词且功能启用时，关键词的每个字符按位置循环映射到 7 种彩虹色。高亮逻辑在 [`claude-code-source/src/components/messages/HighlightedThinkingText.tsx`](../../claude-code-source/src/components/messages/HighlightedThinkingText.tsx#L117-L130) 中实现：

```typescript
// claude-code-source/src/components/messages/HighlightedThinkingText.tsx#L117-L130
parts = [];
let cursor = 0;
for (const t of triggers) {
  if (t.start > cursor) {
    parts.push(<Text key={`plain-${cursor}`} color="text">{text.slice(cursor, t.start)}</Text>);
  }
  for (let i = t.start; i < t.end; i++) {
    parts.push(<Text key={`rb-${i}`} color={getRainbowColor(i - t.start)}>{text[i]}</Text>);
  }
  cursor = t.end;
}
```

`getRainbowColor()` 根据字符索引对 7 取模，返回 `rainbow_red` 到 `rainbow_violet` 之一，使 `ultrathink` 这 10 个字符呈现红→橙→黄→绿→蓝→靛→紫→红→橙→黄的彩虹渐变效果。

### 4. Brief 布局模式

Brief 模式（聊天式紧凑布局）下，组件渲染为标签式结构而非指针式，在 [`claude-code-source/src/components/messages/HighlightedThinkingText.tsx`](../../claude-code-source/src/components/messages/HighlightedThinkingText.tsx#L25-L80) 中定义：

```typescript
// claude-code-source/src/components/messages/HighlightedThinkingText.tsx#L25-L80
if (useBriefLayout) {
  const ts = timestamp ? formatBriefTimestamp(timestamp) : "";
  const t2 = isQueued ? "subtle" : "briefLabelYou";
  // "You" 标签
  t3 = <Text color={t2}>You</Text>;
  // 时间戳（暗淡）
  t4 = ts ? <Text dimColor={true}> {ts}</Text> : null;
  // 正文
  const t6 = isQueued ? "subtle" : "text";
  t7 = <Text color={t6}>{text}</Text>;
  return <Box flexDirection="column" paddingLeft={2}>{t5}{t7}</Box>;
}
```

Brief 模式的视觉特征：

- "You" 标签使用 `briefLabelYou` 颜色（深色主题为 `rgb(122,180,232)` 浅蓝）
- 排队中的消息（`isQueued`）使用 `subtle` 暗淡色，表示尚未发送
- 时间戳使用 `dimColor` 暗淡显示
- 正文使用 `text` 颜色，`paddingLeft={2}` 缩进对齐

## 五、 Brief 布局判断逻辑

### 1. 多条件门控

Brief 布局的启用受多个条件共同控制，在 [`claude-code-source/src/components/messages/UserPromptMessage.tsx`](../../claude-code-source/src/components/messages/UserPromptMessage.tsx#L51-L61) 中计算：

```typescript
// claude-code-source/src/components/messages/UserPromptMessage.tsx#L51-L61
const useBriefLayout = feature('KAIROS') || feature('KAIROS_BRIEF')
  ? (getKairosActive()
      || (getUserMsgOptIn()
          && (briefEnvEnabled
              || getFeatureValue_CACHED_MAY_BE_STALE('tengu_kairos_brief', false))))
    && isBriefOnly
    && !isTranscriptMode
    && !viewingAgentTaskId
  : false;
```

启用条件包括：

- `feature('KAIROS')` 或 `feature('KAIROS_BRIEF')` 编译时门控
- `getKairosActive()` 或用户选择加入 + 环境变量/功能标志
- `isBriefOnly` 全局状态为真
- 非 transcript（转录）模式
- 未在查看 Agent 任务

### 2. Hook 顺序稳定性

由于 `feature()` 是编译时常量，`useAppState`、`useMemo` 等 Hook 被包裹在三元表达式中，外部构建不会包含这些订阅。截断计算（`useMemo`）被提前到 early return 之前，保证 Hook 顺序在所有分支中一致。

## 六、 视觉样式总结

### 1. 默认模式样式

| 元素 | 样式 |
|---|---|
| 容器 | `Box` 列向，`marginTop=1`，背景色 `userMessageBackground`，`paddingRight=1` |
| 指针前缀 | `›` 字符，颜色 `subtle`（未选中）/ `suggestion`（选中） |
| 正文 | `text` 颜色 |
| ultrathink 关键词 | 7 色彩虹循环高亮 |

### 2. Brief 模式样式

| 元素 | 样式 |
|---|---|
| 容器 | `Box` 列向，无背景色，`paddingLeft=2` |
| "You" 标签 | `briefLabelYou` 颜色 |
| 时间戳 | `dimColor` 暗淡 |
| 正文 | `text` 颜色（排队时为 `subtle`） |

### 3. 选中状态样式

选中消息时，容器背景色从 `userMessageBackground` 切换为 `messageActionsBackground`，指针颜色从 `subtle` 切换为 `suggestion`，形成视觉聚焦效果。
