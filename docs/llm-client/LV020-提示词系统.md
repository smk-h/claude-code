## 一、 概述

Claude Code 的提示词系统（Prompt System）是一套分层、模块化的架构，负责从系统提示词构建、工具定义注册、到最终 API 请求发送的完整流程。系统提示词被分为**静态部分**（可缓存，跨会话稳定）和**动态部分**（每轮可能变化，按需计算），通过缓存机制显著降低 Token 消耗。

核心源码文件：

- [`src/constants/prompts.ts`](../../claude-code-source/src/constants/prompts.ts#L444)：系统提示词各 Section 的定义与 `getSystemPrompt()` 主入口
- [`src/utils/systemPrompt.ts`](../../claude-code-source/src/utils/systemPrompt.ts#L41)：`buildEffectiveSystemPrompt()` 优先级合并逻辑
- [`src/constants/systemPromptSections.ts`](../../claude-code-source/src/constants/systemPromptSections.ts#L20)：Section 缓存机制
- [`src/constants/system.ts`](../../claude-code-source/src/constants/system.ts#L10)：CLI 前缀与归属头
- [`src/utils/api.ts`](../../claude-code-source/src/utils/api.ts#L119)：工具 Schema 转换与系统提示词拆分
- [`src/utils/queryContext.ts`](../../claude-code-source/src/utils/queryContext.ts#L44)：`fetchSystemPromptParts()` 提示词组件获取
- [`src/services/api/claude.ts`](../../claude-code-source/src/services/api/claude.ts#L3213)：`buildSystemPromptBlocks()` 最终 API 块构建
- [`src/context.ts`](../../claude-code-source/src/context.ts#L116)：用户上下文与系统上下文

## 二、 提示词组装总流程

从用户输入到最终发送给 LLM 的完整流程如下：

```
用户输入
  → processUserInput() / processTextPrompt()
  → fetchSystemPromptParts()        // 并行获取三部分上下文
     ├── getSystemPrompt()          // 构建默认系统 prompt
     ├── getUserContext()           // 用户上下文: CLAUDE.md、日期
     └── getSystemContext()         // 系统上下文: git 状态、缓存破坏
  → buildEffectiveSystemPrompt()    // 优先级合并: override > coordinator > agent > custom > default
  → queryModelWithStreaming()       // API 调用入口
     → 注入 attribution header + CLI prefix
     → buildSystemPromptBlocks()    // 构建 API 格式系统提示词块
        → splitSysPromptPrefix()   // 按 boundary 拆分为静态/动态块 + 缓存标记
     → toolToAPISchema()           // 每个工具生成 API Schema
     → normalizeMessagesForAPI()   // 消息归一化
     → 发送 API 请求
```

## 三、 系统提示词构成

### 1. getSystemPrompt() 主入口

[`getSystemPrompt()`](../../claude-code-source/src/constants/prompts.ts#L444-L577) 是构建默认系统提示词的主函数，接收 `tools`、`model`、`additionalWorkingDirectories`、`mcpClients` 四个参数，返回 `string[]` 数组。

该函数返回的数组分为**静态内容**（可缓存）和**动态内容**（每轮可能变化）两部分，中间以 `SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 标记分隔：

```typescript
// claude-code-source/src/constants/prompts.ts
return [
  // --- 静态内容 (可缓存, cacheScope: 'global') ---
  getSimpleIntroSection(outputStyleConfig),
  getSimpleSystemSection(),
  outputStyleConfig === null || outputStyleConfig.keepCodingInstructions === true
    ? getSimpleDoingTasksSection()
    : null,
  getActionsSection(),
  getUsingYourToolsSection(enabledTools),
  getSimpleToneAndStyleSection(),
  getOutputEfficiencySection(),
  // === BOUNDARY MARKER - DO NOT MOVE OR REMOVE ===
  ...(shouldUseGlobalCacheScope() ? [SYSTEM_PROMPT_DYNAMIC_BOUNDARY] : []),
  // --- 动态内容 (每轮可能变化) ---
  ...resolvedDynamicSections,
].filter(s => s !== null)
```

### 2. 静态 Section 详解

#### 2.1 Intro Section — 身份定义

[`getSimpleIntroSection()`](../../claude-code-source/src/constants/prompts.ts#L175-L184) 定义了 Agent 的基本身份和核心安全约束：

```
You are an interactive agent that helps users with software engineering tasks.
Use the instructions below and the tools available to you to assist the user.

{CYBER_RISK_INSTRUCTION}  // 安全风险指令
IMPORTANT: You must NEVER generate or guess URLs for the user...
```

根据 `outputStyleConfig` 的配置，身份描述会切换为"按照 Output Style 响应"或"处理软件工程任务"。

#### 2.2 System Section — 系统规则

[`getSimpleSystemSection()`](../../claude-code-source/src/constants/prompts.ts#L186-L197) 包含系统级规则：

- 所有文本输出直接展示给用户，支持 GFM Markdown
- 工具执行在用户选择的权限模式下运行
- `<system-reminder>` 标签说明（系统注入信息，与工具结果无关）
- 外部数据提示注入风险警示
- Hooks 机制说明
- 自动压缩提示（对话不受上下文窗口限制）

#### 2.3 Doing Tasks Section — 任务行为指导

[`getSimpleDoingTasksSection()`](../../claude-code-source/src/constants/prompts.ts#L199-L253) 是最核心的 Section，包含软件工程任务的详细行为指导：

- 任务范围：Bug 修复、功能添加、代码重构、代码解释等
- 代码风格：不做多余事、不猜 URL、不加不必要注释
- 安全编码：防止注入、XSS、SQL 注入等 OWASP Top 10 漏洞
- 交互原则：先读再改、避免文件膨胀、避免向后兼容 hack
- 反馈机制：`/help` 命令与问题报告途径

#### 2.4 Actions Section — 行动谨慎性

[`getActionsSection()`](../../claude-code-source/src/constants/prompts.ts#L255-L267) 规范不可逆操作的确认机制：

- 本地可逆操作（编辑文件、运行测试）可自由执行
- 不可逆操作（删除文件/分支、force push、修改共享基础设施）需确认
- 遇到障碍时，调查根因而非绕过安全检查

#### 2.5 Using Your Tools Section — 工具使用指导

[`getUsingYourToolsSection()`](../../claude-code-source/src/constants/prompts.ts#L269-L314) 指导 LLM 如何选择和使用工具：

- 优先使用专用工具（Read/Edit/Write/Glob/Grep）而非 Bash
- 使用 Task/TodoWrite 工具规划和追踪工作
- 支持并行调用多个无依赖的工具
- REPL 模式下有不同的工具指导

#### 2.6 Tone and Style Section — 语气风格

[`getSimpleToneAndStyleSection()`](../../claude-code-source/src/constants/prompts.ts#L430-L442) 定义输出风格：

- 不使用 emoji
- 响应简洁直接
- 使用反引号格式化文件名、目录名、函数名
- 使用行号引用格式

#### 2.7 Output Efficiency Section — 输出效率

[`getOutputEfficiencySection()`](../../claude-code-source/src/constants/prompts.ts#L403-L428) 规范沟通效率：

- 面向人类写作，而非日志记录
- 关键节点更新（发现 Bug、方向变更、取得进展）
- 使用完整、语法正确的句子
- 根据用户专业水平调整详细程度

### 3. 动态 Section 详解

动态 Section 通过 [`systemPromptSection()`](../../claude-code-source/src/constants/systemPromptSections.ts#L20-L25) 或 [`DANGEROUS_uncachedSystemPromptSection()`](../../claude-code-source/src/constants/systemPromptSections.ts#L32-L38) 创建，支持缓存和每轮重算两种模式。

| Section 名称 | 缓存策略 | 说明 |
|---|---|---|
| `session_guidance` | 缓存 | 会话特定引导（工具使用建议、Agent 指导、Skill 引导等） |
| `memory` | 缓存 | CLAUDE.md / MEMORY.md 内容 |
| `ant_model_override` | 缓存 | Ant 内部模型覆盖指令 |
| `env_info_simple` | 缓存 | 环境信息（OS、Shell、CWD 等） |
| `language` | 缓存 | 语言偏好设置 |
| `output_style` | 缓存 | 输出风格配置 |
| `mcp_instructions` | **每轮重算** | MCP 服务器指令（因 MCP 可能随时连接/断开） |
| `scratchpad` | 缓存 | 临时文件目录说明 |
| `frc` | 缓存 | Function Result Clearing 配置 |
| `summarize_tool_results` | 缓存 | 工具结果摘要指令 |
| `numeric_length_anchors` | 缓存 | 数字化长度锚点（Ant 专属） |
| `token_budget` | 缓存 | Token 预算指令 |
| `brief` | 缓存 | KAIROS 精简模式指令 |

动态 Section 的解析由 [`resolveSystemPromptSections()`](../../claude-code-source/src/constants/systemPromptSections.ts#L43-L58) 统一处理，从缓存中读取或重新计算，缓存通过 [`clearSystemPromptSections()`](../../claude-code-source/src/constants/systemPromptSections.ts#L65-L68) 在 `/clear` 或 `/compact` 时清除。

### 4. Proactive 模式特殊路径

当 Proactive 或 KAIROS 特性启用时，系统提示词走一条精简路径（[行 466-489](../../claude-code-source/src/constants/prompts.ts#L466-L489)）：

```typescript
return [
  `You are an autonomous agent. Use the available tools to do useful work.
${CYBER_RISK_INSTRUCTION}`,
  getSystemRemindersSection(),
  await loadMemoryPrompt(),
  envInfo,
  getLanguageSection(settings.language),
  isMcpInstructionsDeltaEnabled() ? null : getMcpInstructionsSection(mcpClients),
  getScratchpadInstructions(),
  getFunctionResultClearingSection(model),
  SUMMARIZE_TOOL_RESULTS_SECTION,
  getProactiveSection(),
].filter(s => s !== null)
```

此路径跳过了大部分静态 Section，以更精简的 prompt 支持自主代理模式。

## 四、 系统提示词优先级合并

[`buildEffectiveSystemPrompt()`](../../claude-code-source/src/utils/systemPrompt.ts#L41-L123) 负责决定最终使用哪个系统提示词，按以下优先级链处理：

### 1. 优先级从高到低

| 优先级 | 来源 | 行为 |
|---|---|---|
| 0 | **Override** | Loop/REPL 模式覆盖一切，直接返回 |
| 1 | **Coordinator** | 协调者模式使用 `getCoordinatorSystemPrompt()` |
| 2 | **Agent** | 主线程代理定义的 prompt |
| 3 | **Custom** | `--system-prompt` 参数指定 |
| 4 | **Default** | 标准的 `getSystemPrompt()` 结果 |
| 追加 | **Append** | `appendSystemPrompt` 始终追加（Override 时除外） |

### 2. Agent Prompt 的特殊处理

在 Proactive 模式下，Agent prompt 是**追加**到默认 prompt 之后的：

```typescript
// claude-code-source/src/utils/systemPrompt.ts#L103-L113
if (agentSystemPrompt && isProactiveActive_SAFE_TO_CALL_ANYWHERE()) {
  return asSystemPrompt([
    ...defaultSystemPrompt,
    `\n# Custom Agent Instructions\n${agentSystemPrompt}`,
    ...(appendSystemPrompt ? [appendSystemPrompt] : []),
  ])
}
```

其他模式下，Agent prompt **替换**默认 prompt：

```typescript
// claude-code-source/src/utils/systemPrompt.ts#L115-L122
return asSystemPrompt([
  ...(agentSystemPrompt
    ? [agentSystemPrompt]
    : customSystemPrompt
      ? [customSystemPrompt]
      : defaultSystemPrompt),
  ...(appendSystemPrompt ? [appendSystemPrompt] : []),
])
```

## 五、 CLI 前缀与归属头

### 1. CLI 前缀

[`getCLISyspromptPrefix()`](../../claude-code-source/src/constants/system.ts#L30-L46) 根据运行环境返回三种身份前缀：

| 前缀 | 场景 |
|---|---|
| `You are Claude Code, Anthropic's official CLI for Claude.` | 默认交互模式 |
| `You are Claude Code, Anthropic's official CLI for Claude, running within the Claude Agent SDK.` | SDK 非交互模式 + 有 appendSystemPrompt |
| `You are a Claude agent, built on Anthropic's Claude Agent SDK.` | SDK 非交互模式 |

### 2. 归属头

[`getAttributionHeader()`](../../claude-code-source/src/constants/system.ts#L73-L95) 在系统提示词最前面注入 `x-anthropic-billing-header`，包含：

- `cc_version`：版本号 + 指纹
- `cc_entrypoint`：入口点标识
- `cch`：客户端认证哈希（NATIVE_CLIENT_ATTESTATION 启用时）
- `cc_workload`：工作负载标识（用于 QoS 路由）

归属头不参与缓存标记（`cacheScope: null`）。

## 六、 系统提示词 API 组装

### 1. splitSysPromptPrefix() — 拆分与缓存标记

[`splitSysPromptPrefix()`](../../claude-code-source/src/utils/api.ts#L321-L399) 将系统提示词字符串数组拆分为 `SystemPromptBlock[]`，每个块携带 `cacheScope`：

**启用 Global Cache Scope 时（默认路径）：**

```
[0] { text: attributionHeader,      cacheScope: null }    // 归属头不缓存
[1] { text: CLISyspromptPrefix,     cacheScope: null }    // 前缀不缓存
[2] { text: 静态Section合并文本,     cacheScope: 'global' } // 静态内容全局缓存
[3] { text: 动态Section合并文本,     cacheScope: null }    // 动态内容不缓存
```

静态内容和动态内容以 `SYSTEM_PROMPT_DYNAMIC_BOUNDARY` 为界，boundary 前的块合并为一个 `global` 缓存块，boundary 后的合并为无缓存块。

**Tool-based Cache 路径**（`skipGlobalCacheForSystemPrompt` 为 true 时）：

```
[0] { text: attributionHeader,  cacheScope: null }
[1] { text: CLISyspromptPrefix, cacheScope: 'org' }
[2] { text: 其余内容合并,        cacheScope: 'org' }
```

### 2. buildSystemPromptBlocks() — 最终 API 格式

[`buildSystemPromptBlocks()`](../../claude-code-source/src/services/api/claude.ts#L3213-L3237) 在 `splitSysPromptPrefix()` 的基础上，将每个 `SystemPromptBlock` 转换为 Anthropic API 所需的 `TextBlockParam[]`，并在启用 Prompt Caching 时添加 `cache_control` 字段：

```typescript
// claude-code-source/src/services/api/claude.ts
export function buildSystemPromptBlocks(
  systemPrompt: SystemPrompt,
  enablePromptCaching: boolean,
  options?: {
    skipGlobalCacheForSystemPrompt?: boolean
    querySource?: QuerySource
  },
): TextBlockParam[] {
  return splitSysPromptPrefix(systemPrompt, {
    skipGlobalCacheForSystemPrompt: options?.skipGlobalCacheForSystemPrompt,
  }).map(block => {
    return {
      type: 'text' as const,
      text: block.text,
      ...(enablePromptCaching &&
        block.cacheScope !== null && {
          cache_control: getCacheControl({
            scope: block.cacheScope,
            querySource: options?.querySource,
          }),
        }),
    }
  })
}
```

## 七、 上下文注入

### 1. 用户上下文 — getUserContext()

[`getUserContext()`](../../claude-code-source/src/context.ts#L155-L189) 提供用户侧上下文信息：

- `claudeMd`：CLAUDE.md 文件内容（目录遍历自动发现）
- `currentDate`：当前日期

CLAUDE.md 的加载逻辑：
- `CLAUDE_CODE_DISABLE_CLAUDE_MDS` 环境变量可硬关闭

### 3. 工具 Schema 转换

[`toolToAPISchema()`](../../claude-code-source/src/utils/api.ts#L119) 将内部 `Tool` 定义转换为 Anthropic API 所需的 `BetaToolUnion` 格式：

```typescript
// claude-code-source/src/utils/api.ts#L119-L135
export async function toolToAPISchema(
  tool: Tool,
  options: {
    getToolPermissionContext: () => Promise<ToolPermissionContext>
    tools: Tools
    agents: AgentDefinition[]
    allowedAgentTypes?: string[]
    model?: string
    deferLoading?: boolean
    cacheControl?: { type: 'ephemeral'; scope?: 'global' | 'org'; ttl?: '5m' | '1h' }
  },
): Promise<BetaToolUnion>
```

转换流程：
1. 检查 `toolSchemaCache` 中是否已有缓存
2. 优先使用 `inputJSONSchema`（MCP 工具原始 Schema），否则用 `zodToJsonSchema()` 转换 Zod Schema
3. 过滤 Swarm 相关字段（`filterSwarmFieldsFromSchema()`）
4. 组装 `name`、`description`（来自 `tool.prompt()`）、`input_schema`
5. 按条件添加 `strict`、`eager_input_streaming`、`defer_loading`、`cache_control` 字段

### 4. MCP 工具集成

MCP（Model Context Protocol）工具通过 MCP 服务器动态注册，关键特性：
- MCP 指令通过 [`getMcpInstructionsSection()`](../../claude-code-source/src/constants/prompts.ts#L579) 注入系统提示词
- MCP 指令使用 `DANGEROUS_uncachedSystemPromptSection`（每轮重算），因为 MCP 服务器可能随时连接/断开
- 启用 `MCP Instructions Delta` 时，改为通过附件注入以避免破坏 Prompt 缓存
- MCP 工具的 Schema 直接使用 `inputJSONSchema`，不经过 Zod 转换

## 十、 上下文窗口与压缩机制

### 1. 上下文窗口大小

[`getContextWindowForModel()`](../../claude-code-source/src/utils/context.ts#L51-L98) 按以下优先级确定上下文窗口大小：

| 优先级 | 来源 | 说明 |
|---|---|---|
| 1 | `CLAUDE_CODE_MAX_CONTEXT_TOKENS` | 环境变量覆盖（Ant 专属） |
| 2 | `[1m]` 后缀检测 | 模型名含 `[1m]` → 1,000,000 tokens |
| 3 | 模型能力查询 | `getModelCapability(model).max_input_tokens` |
| 4 | 1M Beta 头 | 支持 1M 的模型 + Beta 头 |
| 5 | Sonnet 1M 实验 | `coral_reef_sonnet` 全局配置 |
| 6 | Ant 模型配置 | `resolveAntModel(model).contextWindow` |
| 7 | 默认值 | 200,000 tokens |

### 2. 自动压缩

[`autoCompactIfNeeded()`](../../claude-code-source/src/services/compact/autoCompact.ts) 在上下文接近窗口限制时自动触发压缩：

- [`getEffectiveContextWindowSize()`](../../claude-code-source/src/services/compact/autoCompact.ts#L33-L49)：上下文窗口减去摘要输出保留 Token（最多 20,000）
- 阈值计算：`effectiveContextWindow - 13,000`（AUTOCOMPACT_BUFFER_TOKENS）
- 压缩方式：`compactConversation()` 生成对话摘要
- 连续失败上限：3 次（避免浪费 API 调用）

### 3. 其他压缩策略

| 策略 | 说明 |
|---|---|
| **Snip 压缩** | `snipCompactIfNeeded()` 裁剪历史消息 |
| **微压缩** | `microcompact()` 缓存编辑式压缩 |
| **上下文折叠** | `applyCollapsesIfNeeded()` 折叠低价值消息 |
| **工具结果预算** | `applyToolResultBudget()` 截断大型工具结果 |
| **Function Result Clearing** | 自动清除旧工具结果，保留最近 N 个 |
